# CustomNode 重构踩坑记录

> 将 mini-canvas 的自定义节点从 scoped CSS（深色主题）重构为 Tailwind CSS（亮色主题）时遇到的问题。

---

## 错误 1：Template 多根节点导致 Vue 警告

**现象**：控制台反复输出：

```
[Vue warn]: Extraneous non-emits event listeners (updateNodeInternals) were passed to component
but could not be automatically inherited because component renders fragment or text root nodes.
```

**根因**：VueFlow 会给自定义节点组件传入 `updateNodeInternals` 等事件监听器。当 `<template>` 有多个根节点（fragment）时，Vue 无法自动继承这些事件监听器到具体 DOM 元素上。

**错误写法**：

```html
<template>
  <!-- 根节点 1：外层 relative 容器 -->
  <div class="relative pt-5">...</div>
  <!-- 以下内容如果没有被外层包裹，就会形成 fragment -->
</template>
```

**修复**：确保 `<template>` 内有且只有一个根元素。

```html
<template>
  <div class="relative">  <!-- 唯一根节点 -->
    <!-- 所有内容在这里 -->
  </div>
</template>
```

**教训**：VueFlow 自定义节点组件必须是单根节点，不能是 fragment。

## 错误 2：文档流内放置装饰元素导致节点边界计算错误

**现象**：VueFlow 计算出的节点尺寸为 320×256，而不是期望的卡片尺寸 256×256。

**根因**：VueFlow 通过测量节点根元素的边界框（bounding box）来确定节点尺寸。左右加号按钮如果放在 flex 流式布局中，它们的尺寸会被一并计入节点的宽高。

**错误写法**：

```html
<div class="relative">
  <div class="flex items-center gap-3">
    <button>+</button>                    <!-- ← 计入宽度 -->
    <div class="w-64 h-64">卡片</div>     <!-- 期望的 256×256 -->
    <button>+</button>                    <!-- ← 计入宽度 -->
  </div>
</div>
```

此时根元素实际尺寸 = 按钮宽度 + gap + 卡片宽度 + gap + 按钮宽度 ≈ 320

**正确写法**：装饰元素（按钮、标题等不影响节点核心区域的元素）必须用 `absolute` 定位脱离文档流。

```html
<div class="relative">                    <!-- 尺寸 = 卡片 = 256×256 -->
  <!-- 标题：absolute，飘在上方 -->
  <div class="absolute -top-5 ...">...</div>

  <!-- 左按钮：absolute，定位到卡片左侧 -->
  <button class="absolute left-0 -translate-x-[calc(100%+12px)] ...">+</button>

  <!-- 卡片：唯一的流内元素 -->
  <div class="w-64 h-64">卡片</div>

  <!-- 右按钮：absolute，定位到卡片右侧 -->
  <button class="absolute right-0 translate-x-[calc(100%+12px)] ...">+</button>
</div>
```

**教训**：节点的核心区域决定节点边界。标题、工具栏、装饰按钮等必须用绝对定位放在核心区域之外。
