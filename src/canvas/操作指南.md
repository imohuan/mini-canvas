# Canvas 操作指南

## 节点端口（输入 / 输出）

每个节点通过 `sourcePosition` / `targetPosition` 定义端口：

| 属性 | 位置 | 含义 |
|---|---|---|
| `sourcePosition: Position.Right` | 右侧 | **输出端口**，只能从此处拖出连线 |
| `targetPosition: Position.Left` | 左侧 | **输入端口**，只能从此处接收连线 |
| 仅设 source | 仅右侧 | **纯输出节点**（如 Node A） |
| 仅设 target | 仅左侧 | **纯输入节点**（如 Node C） |
| 两属性都设 | 左右都有 | **输入+输出节点**（如 Node B） |

> ⚠️ **连线规则：只能从「输出端口（右侧）」→「输入端口（左侧）」**  
> 由 `isValidConnection` 函数拦截：源节点无 `sourcePosition` 或目标节点无 `targetPosition` → 拒绝连线并 Console 警告。
>
> **实现方式：** 通过 `<template #node-default>` 插槽自定义节点渲染，直接渲染 `Handle` 组件在左/右位置，确保端口位置不受默认节点逻辑影响。

**当前节点配置：**

```ts
// Node A → 纯输出
{ id: '1', sourcePosition: Position.Right }
// Node B → 输入+输出
{ id: '2', sourcePosition: Position.Right, targetPosition: Position.Left }
// Node C → 纯输入
{ id: '3', targetPosition: Position.Left }
```

合法连线：A→B ✓   A→C ✓   B→C ✓   B→A ✗   C→A ✗   C→B ✗

## 连接线样式系统

### 设计目标

通过 Pannel 控制面板实时调整连接线的**宽度、颜色、线型（流动/直线/阶梯）、虚实（实线/虚线）**，改动即时应用到画布上的所有边。

### 架构全景

```
Pannel.vue ──(emit)──> Canvas.vue ──(write)──> useCanvasState Store (Pinia)
                                                      │
                                                      │ (read, reactive)
                                                      ▼
                                              CustomEdge.vue
```

核心思路：**连接线样式不经过 VueFlow 的内部状态系统**，而是通过 Pinia 全局 store 在 Pannel 和 CustomEdge 之间直连。

### 实现文件

| 文件 | 职责 |
|---|---|
| `useCanvasState.ts` | Pinia setup store，存储所有可配置状态（连接线样式 + 节点/视口交互）|
| `Pannel.vue` | 控制面板 UI：线型按钮组、宽度滑块、颜色预设选择器、虚实切换 |
| `Canvas.vue` | 绑定 Pannel 的 `v-model` / 事件到 Pinia store，生成初始 edge data |
| `CustomEdge.vue` | 读取 Pinia store 样式值，渲染 SVG path + 流光效果 + 剪切按钮 |

---

### 1. 状态存储：`useCanvasState.ts`

```ts
export const useCanvasStore = defineStore('canvasState', () => {
  /** 连接线宽度，单位 px，范围 1~6 */
  const edgeLineWidth = ref(2)
  /** 连接线颜色，hex 格式例如 #3b82f6 */
  const edgeColor = ref('#3b82f6')
  /** 连接线类型：flowing | straight | step */
  const edgeType = ref<EdgeType>('flowing')
  /** 连接线虚实：false=实线，true=虚线 */
  const edgeDashed = ref(false)
  // ... 还有 connectionMode、nodesDraggable 等 13 个交互状态
})
```

所有属性都是 `ref()`，Pinia 自动将其 unwrap 为响应式属性。任何组件调用 `useCanvasStore()` 拿到的都是同一个单例。

---

### 2. 控制面板：`Pannel.vue`

面板的"连接线样式"区域提供 4 个控件：

| 控件 | 交互方式 | 向上通知 |
|---|---|---|
| 线型选择（流动/直线/阶梯）| 按钮组点击 | `emit('update:edgeType', value)` |
| 宽度滑块（1~6px）| range 滑块拖拽 | `emit('update:edgeLineWidth', value)` |
| 颜色预设（7 色圆点 + 原生选色器）| 点击/选色 | `emit('update:edgeColor', value)` |
| 虚实切换 | 按钮点击 | `emit('toggleEdgeDashed')` |

前三个使用 Vue `v-model:xxx` 模式（prop + `update:xxx` emit），通过 Canvas 绑到 Pinia store：

```html
<!-- Canvas.vue -->
<Pannel
  v-model:edgeLineWidth="canvas.edgeLineWidth"
  v-model:edgeColor="canvas.edgeColor"
  v-model:edgeType="canvas.edgeType"
  ...
/>
```

虚实切换**不使用** `v-model`，而是发 `toggleEdgeDashed` 事件，Canvas 直接翻转 store：

```html
<!-- Canvas.vue -->
<Pannel :edgeDashed="canvas.edgeDashed" @toggle-edge-dashed="canvas.edgeDashed = !canvas.edgeDashed" />
```

---

### 3. 边渲染：`CustomEdge.vue`

CustomEdge 直接读 Pinia store，不依赖 props.data：

```ts
const canvas = useCanvasStore()

const edgeType = computed(() => canvas.edgeType as 'flowing' | 'straight' | 'step' || 'flowing')
const lineWidth = computed(() => canvas.edgeLineWidth)
const edgeColor = computed(() => canvas.edgeColor)
const dashArray = computed(() => canvas.edgeDashed ? `${lineWidth.value * 4} ${lineWidth.value * 2}` : undefined)
```

**关键点：直接在 computed 内读 `canvas.edgeDashed`，不用 `storeToRefs` 解构。**

所有 SVG `<path>` 都绑定这些 computed 值：

```html
<path :stroke="edgeColor" :stroke-width="lineWidth" :stroke-dasharray="dashArray" ... />
```

当 Pinia store 中的值变化时，`computed` 自动重算，SVG 立即重渲染。**不需要 watch、不需要 `updateEdge()`、不需要替换 edges 数组。**

---

### 4. 流光效果

从主项目 `FlowingEdge.vue` 复刻：

- **3 种线型**：`flowing`（贝塞尔曲线）、`straight`（直线）、`step`（阶梯线），通过 `edgePath` computed 动态计算
- **选中高亮**：`isHighlighted = edge.selected || 任一端点节点被选中`
- **流动动画**：`flowing` 线型 + 高亮态时，`requestAnimationFrame` 驱动渐变偏移 + `<animateMotion>` 双粒子
- **剪切按钮**：双击连接线弹出圆形 ✕ 按钮，调用 `removeEdges` 删除

## 踩坑记录：虚线始终显示（`edgeDashed=false` 却仍是虚线）

### Bug 现象

Pinia DevTools 显示 `edgeDashed: false`，Pannel 按钮显示"实线"，但画布上的连线始终是虚线。颜色、宽度等属性修改正常生效，唯独虚线/实线切换无效。

### 根因 ⚠️：Vue Flow `animated: true` CSS 强制覆盖

**这是导致虚线的根因（而非 Pinia 响应性问题）。**

```ts
// Canvas.vue — 原代码
const edges = ref<Edge[]>([
  { id: 'e1-2', type: 'custom', source: '1', target: '2', animated: true, data: ... },
  { id: 'e2-3', type: 'custom', source: '2', target: '3', animated: true, data: ... },
])
```

当 edge 上设置 `animated: true` 时，Vue Flow 会给 edge 的 DOM wrapper 添加 CSS class `animated`。其内置样式表（`@vue-flow/core/dist/style.css`）包含：

```css
.animated path {
  stroke-dasharray: 5;
  animation: dashdraw .5s linear infinite;
}
```

**这意味着：** 无论 CustomEdge 组件在 SVG `<path>` 上绑定了什么 `:stroke-dasharray="dashArray"`，Vue Flow 的全局 CSS 规则 `.animated path { stroke-dasharray: 5 }` 都会以**更高优先级**覆盖它，导致所有线永远是虚线。

### Pinia 响应性也参与了问题（次要）

```ts
// CustomEdge.vue — 原代码
const { edgeDashed: stDashed } = storeToRefs(canvas)
const dashArray = computed(() => stDashed.value ? '...' : null)
```

两个子问题叠加：

| 问题 | 详情 |
|---|---|
| `storeToRefs` 解构 | `stDashed` 是在 computed 外部解构的 Pinia ref。在 computed 内读 `.value` 时，Vue 的依赖追踪链在某些场景（Pinia Proxy + VueFlow Teleport + computed 嵌套）下会断链，`edgeDashed` 变化后 computed 不重算 |
| `null` → SVG 字面量 | `:stroke-dasharray="null"` 在 Vue 3 中渲染为 `stroke-dasharray="null"` 字符串，不同浏览器处理不一致 |

### 三层修复

**第 1 层（根因）：移除 `animated: true`**

```ts
// Canvas.vue — 修复后
const edges = ref<Edge[]>([
  { id: 'e1-2', type: 'custom', source: '1', target: '2', data: makeEdgeData() },
  { id: 'e2-3', type: 'custom', source: '2', target: '3', data: makeEdgeData() },
])
```

CustomEdge 已有自己的流光动画（`<animateMotion>` + 渐变），不需要 Vue Flow 的 `animated`。

**第 2 层：弃用 `storeToRefs`，改直接读 store**

```ts
// CustomEdge.vue — 修复后
const canvas = useCanvasStore()
const dashArray = computed(() => canvas.edgeDashed
  ? `${lineWidth.value * 4} ${lineWidth.value * 2}`
  : undefined)
```

直接在 computed 内读 `canvas.edgeDashed`（store 实例本身是响应式 Proxy），Vue 能正确建立依赖追踪链。

**第 3 层：`null` → `undefined`**

```ts
// 修复前
:stroke-dasharray="null"   // → 渲染为 stroke-dasharray="null"
// 修复后
:stroke-dasharray="undefined"  // → 不渲染该属性（即实线）
```

Vue 3 遇到 `undefined` 的绑定值时不输出该属性，遇到 `null` 会输出字面量 `"null"`。

### 事后分析：为什么颜色/宽度能改但虚线不行？

颜色、宽度通过同一个 `storeToRefs` 模式读写，但它们**碰巧能工作**，因为：

- 颜色、宽度在初始渲染时 computed 已正确建立依赖链（`edgeColor.value`、`edgeLineWidth.value` 在 computed 首次求值时被正确追踪）
- `edgeDashed` 的 computed 在某些时序下没能建立正确的依赖，所以后续变化不被追踪
- 但即使 computed 正确了，Vue Flow 的 `.animated path { stroke-dasharray: 5 }` CSS 还是会覆盖，所以**根因始终是 `animated: true`**

## 踩坑记录：实时样式修改

### 尝试 1：watch + 直接改 `e.data` → 失败 ❌

```ts
// Canvas.vue
watch([edgeLineWidth, edgeColor, edgeType], () => {
  edges.value.forEach(e => { e.data = { ...e.data, ...newStyles } })
})
```

**原因**：VueFlow 内部维护自己的 edge 状态副本。直接 mutate `edges.value` 数组中对象的 `data` 属性，VueFlow 的浅比较检测不到变化，不会触发 CustomEdge 重渲染。

---

### 尝试 2：`edges.value = edges.value.map(...)` → 失败 ❌

```ts
edges.value = edges.value.map(e => ({ ...e, data: { ...e.data, ...newStyles } }))
```

**原因**：虽然替换了数组引用，但 VueFlow 同步新 edge 对象时，CustomEdge 可能因为 key 相同而被 Vue 复用，`props.data` 没有收到新的对象引用，computed 不重算。VueFlow 的 props 传递机制不保证深层 data 变更一定触发 re-render。

---

### 尝试 3：`updateEdge(id, { data: {...} })` → 失败 ❌

```ts
const { updateEdge } = useVueFlow()
updateEdge(e.id, { ...e, data: { ...e.data, ...newStyles } })
```

**原因**：`updateEdge()` 能触发 VueFlow 内部状态更新，但同样受限于 VueFlow props 传递机制。部分情况下 Vue 组件复用导致 data 变更不被感知。

---

### 尝试 4：`provide/inject` → 失败 ❌

```ts
// Canvas.vue
provide('edgeColor', edgeColor)

// CustomEdge.vue
const color = inject('edgeColor')
```

**原因**：**这是最关键的发现。** VueFlow 使用 `<Teleport>` 将 edge 组件渲染到 SVG viewport 层。Teleport 虽然不改变组件树结构（provide/inject 理论上穿透 Teleport），但 VueFlow 内部的 SVG 层级使用了额外的组件包装，**切断了 provide 的祖先链**。CustomEdge 的 `inject()` 拿到 `undefined`，fallback 到 props.data 的静态初始值。

---

### 尝试 5：共享 `reactive()` 模块 → 失败 ❌

```ts
// edgeStyleConfig.ts
export const config = reactive({ color: '#3b82f6', ... })

// Canvas.vue + CustomEdge.vue 都 import 同一个实例
```

**原因**：Vite 的模块打包 + Pinia 的 `@vue/reactivity` 可能会导致模块实例在某些情况下不共享（特别是 HMR 热更新时）。两个组件拿到的 `config` 对象不是同一个引用，改动无法跨组件传递。

---

### 最终方案：Pinia Store（直接读，不用 `storeToRefs`）✅

```ts
// CustomEdge.vue — 最终写法
const canvas = useCanvasStore()

const edgeType = computed(() => canvas.edgeType)
const lineWidth = computed(() => canvas.edgeLineWidth)
const edgeColor = computed(() => canvas.edgeColor)
const dashArray = computed(() => canvas.edgeDashed
  ? `${lineWidth.value * 4} ${lineWidth.value * 2}`
  : undefined)
```

**为什么有效**：

1. **Pinia 是真正的全局单例** — 同一个 store instance，不依赖 provide/inject 链，不依赖模块共享
2. **直接读 `canvas.xxx`（不用 `storeToRefs`）** — Pinia store 实例本身就是响应式对象，在 computed 内访问其属性能正确建立依赖追踪链。`storeToRefs` 解构后再在 computed 内读 `.value`，Vue 的依赖追踪在某些场景下（Pinia + Teleport + computed 嵌套）会断链，导致 `edgeDashed` 变化时 computed 不重算
3. **`dashArray: undefined` 移除属性** — 虚线为 `false` 时返回 `undefined`（而非 `null` 或 `'0'`），Vue 3 模板引擎会将 `stroke-dasharray` 属性从 SVG 元素上完全移除。若返回 `null`，Vue 会渲染成 `stroke-dasharray="null"` 字面量，不同浏览器处理行为不一致

### 虚线切换特殊处理

`edgeDashed`（boolean）与前三个属性（number/string）不同：Pannel 不使用 `v-model:edgeDashed`，改为：

```html
<!-- Pannel -->
@click="emit('toggleEdgeDashed')"

<!-- Canvas -->
@toggle-edge-dashed="canvas.edgeDashed = !canvas.edgeDashed"
```

**原因**：`v-model` 模式下 Pannel 通过 `!edgeDashed` 计算新值再 emit，依赖 prop 值是"当前最新值"。但由于 VueFlow 的 Teleport 和 Pinia 的 Proxy 层级叠加，prop 可能在某些时序下读到旧值，导致连续点击切换两次又回到原值。改为 toggle 事件后，Canvas 直接从 store 读取当前值并翻转，消除了 prop 抖动的可能性。
