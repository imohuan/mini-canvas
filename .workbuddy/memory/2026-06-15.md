## 第二轮修复（三个问题）

### 1. 多选复制粘贴失灵
- 根因1：边映射 `idMap.get(e.source)` 在大量节点时可能返回 undefined，添加双重验证（`sourceExists`/`targetExists`）和详细日志
- 根因2：粘贴后未保持多选状态 → 粘贴完成后调用 `msApi.clearSelection()` + 逐个添加新节点 ID 到 `selectedNodeIds`
- 添加每条边的映射日志 `[clipboard] Paste: 边映射 e1-2 → xxx, node-1→xxx, node-2→xxx`

### 2. HistoryPlugin 重做 UI 不刷新
- 根因：`context.actions.updateNode(id, { position })` 在 redo 中调用时，VueFlow 内部可能去重（位置值相同对象），导致 UI 不更新
- 修复：undo/redo 回调中**直接修改 reactive node.position.x/y**（`const n = getNodes().find(); n.position.x = newX; n.position.y = newY`），绕过 VueFlow 的 updateNode 去重逻辑
- 始末位置存储为独立对象（`sPos = {x, y}`, `ePos = {x, y}`），不依赖闭包内的 reactive 引用

### 3. 右上角不显示插件列表 + 配置改为响应式
- 根因：`installedPluginNames` 是普通数组（`const arr = []`），Vue 无法追踪变化，computed `loadedPlugins` 不更新
- 修复：改为 `ref<string[]>([])`，确保 Panel 的 `v-for` 响应式渲染
- 新增 `pluginConfigs` prop（`Record<string, Record<string, unknown>>`），支持从外部传入响应式插件配置
- 新增 `pluginApis` reactive map，在 install 后收集所有插件 API
- 所有插件添加完整的 `/** */` JSDoc 注释（类型说明、事件说明、使用示例）
- 所有插件添加 `context.logger` 分级日志（debug/info/warn/error）用于调试

## 第三轮修复（对齐辅助线 + 多选背景框）

### AlignGuidePlugin 坐标修复
- 根因：SVG 内嵌 viewport 方式，viewport 的 CSS transform 叠加导致坐标映射错误
- 参考 `HelperLines.vue` 的正确做法：在外层 DOM 手动计算 transform 公式
- 公式：`screenX = linePosition * zoom + viewportX`，`screenY = linePosition * zoom + viewportY`
- 辅助线容器渲染在 `.canvas-container` 中（viewport 外层），绝对定位
- 每次渲染重新计算 viewport 状态，确保缩放/平移后坐标正确

### 新增 SelectionFrame.vue 多选背景框
- 参考 `src/modules/canvas/components/common/SelectionFrame.vue`
- 2+ 节点选中时自动显示虚线边框包裹所有选中节点
- 支持批量拖拽（直接修改 reactive node.position）
- 支持中键平移、滚轮转发
- 使用 viewport transform wrapper（与 VueFlow 内置 NodesSelection 一致）

### MultiSelectPlugin 改用 VueFlow 内置选中
- 不再自定义 selectedNodeIds/selectedEdgeIds 管理
- 不再禁用 elementsSelectable
- 使用 VueFlow 的 `n.selected`（Ctrl+点击多选、Shift+点击追加、框选自动选中）
- 框选：保留 paneMouseDown/Move/Up 实现自定义选择框（屏幕坐标）
- SelectionFrame 在 Canvas.vue 中渲染，通过 ref 同步 nodes 数据
- ClipboardPlugin 简化为使用 `n.selected` 获取选中节点（移除 MultiSelectAPI 依赖）
