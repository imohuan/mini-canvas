## 修复 ProseMirror 编辑器复制到外部时内容变成空换行

**文件**: `packages/prosemirror-editor-bundle/src/useEditor.ts`

**问题**: 复制编辑器内容粘贴到其他输入框/软件时，粘贴结果变成空换行，而非 `@xxx 文本` 格式。

**根因**: `clipboardTextSerializer` 使用 `slice.content.forEach()` 只遍历顶层节点。文档结构 `doc > paragraph > [text, resource, text]`，顶层全是 paragraph（block），`forEach` 遇到 block 只追加 `\n`，段落内部的 text 和 resource 节点从未被访问。

**修复**: 写了一个递归 `collect(fragment)` 函数，对 block 节点深入 `collect(node.content)` 遍历子节点，遍历完后再追加换行。末尾 `trim()` 清理首尾空白。`getPlainText()` 用 `doc.descendants()` 能正确递归，但 Slice 的 Fragment 没有 `descendants()`，所以手动递归。

## ProseMirror 编辑器 Ctrl+V 粘贴事件冒泡修复

- **问题**: 编辑器内 Ctrl+V 触发了父级/document 上的 paste 事件
- **根因**: `@keydown.stop` 只拦截 keydown 事件，`paste` 是独立的 ClipboardEvent，不受 keydown 管辖。`FileDropPlugin.ts` 在 `document` 上注册了全局 paste 监听器，编辑器内 paste 会冒泡触发
- **修复**: `index.vue` wrapper div 新增 `@paste.stop @cut.stop`（cut 同理，虽 Ctrl+X 的 keydown 已被拦截，但 cut ClipboardEvent 也需防御）
