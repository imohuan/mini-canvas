## 修复：拖拽连接线时 BaseNode 递归更新错误

**错误**：`Maximum recursive updates exceeded in component <BaseNode>` — 拖拽连接线时 Vue 检测到递归更新超过上限。

**根因**：`useCanvasConnection.ts` 的 `buildConnectionEdgeProps` 函数在 Vue render 阶段写入 `canvas.connectionState.hoverNode`（Pinia reactive state）。该函数在 `<template #connection-line>` 插槽中调用，属于 render 阶段。写入触发的 reactive 依赖传播回自身，形成 `render → write → re-render → ...` 递归链。

**修复（两处）**：

1. **`useCanvasConnection.ts:780-794`** — 将 `hoverNode` 写入包装在 `nextTick` 中，迁出 render 阶段：
   ```ts
   if (hoverChanged) {
     const nextHover = ... // 捕获快照值
     nextTick(() => { canvas.connectionState.hoverNode = nextHover })
   }
   ```

2. **`BaseNode.vue:273-306`** — `cardInlineStyle` 从 `computed`（每次返回新对象）改为 `shallowRef` + `watch`（值变化时才更新引用），减少不必要的样式重绘。放置位置在 `cardTransform` 定义之后，确保依赖的 `cardWidth`、`cardHeight`、`zoom`、`cardTransform` 已声明。

## 实现：图片节点蒙版绘制功能

完全参照裁剪模式的 `_overlay` + `group` 分组按钮架构。

**新增文件：**
- `src/canvas/core/nodes/image/ImageMasker.vue` — 蒙版绘制 overlay 组件，Teleport to body，双 canvas（背景图 + 绘制层），支持画笔/橡皮擦双模式

**修改文件：**
- `src/canvas/core/types/CanvasNodeData.ts` — 新增 `MaskConfig` 接口（brushSize/brushColor/brushOpacity/isErasing），`CanvasNodeOverlay` 新增 `_maskMode`、`_maskConfig`
- `src/canvas/core/plugins/storage/sanitizeForSave.ts` — RUNTIME_FIELDS 新增 `maskUrl`、`_maskMode`、`_maskConfig`
- `src/canvas/core/nodes/image/ImageNode.vue` — 集成 ImageMasker，新增 `isMasking`/`maskConfig` computed，`onMaskUpdate` 回调
- `src/canvas/core/nodes/image/index.ts` — 导出 ImageMasker
- `src/canvas/core/nodes/image/ImageNodePlugin.ts` — 新增 6 个命令（mask/maskConfirm/maskCancel/maskConfig/maskClear/maskEraser），注册 mask group 工具栏按钮（画笔大小/颜色/橡皮擦/清除/确认/取消），以及 default group 入口按钮

**工具栏按钮布局（group: 'mask'）：**
1. 画笔大小下拉（5/10/20/40/60px）→ `image.maskConfig` with filter
2. 画笔颜色下拉（红/绿/蓝/黄/白/黑）→ `image.maskConfig` with filter
3. 橡皮擦切换 → `image.maskEraser`
4. 清除蒙版（danger）→ `image.maskClear`
5. 完成 → `image.maskConfirm`（持久化蒙版到 assetManager）
6. 取消（danger）→ `image.maskCancel`（丢弃蒙版）

**技术决策：**
- 蒙版数据通过 `maskUrl`（blob URL）存储在 node.data 顶层，_overlay 仅存 _maskConfig 运行时配置
- 橡皮擦使用 Canvas `destination-out` composite 实现
- 画笔颜色选择时自动关闭橡皮擦模式
- ImageMasker 通过 `maskDataUrl` prop + watcher 响应外部清除操作

### 修复：蒙版坐标计算错误 + 画笔 slider 改造

**坐标计算根因**：`clientToCanvas` 多除了 `d.scale`（`dw/iw`）。Canvas CSS 尺寸和渲染分辨率都等于 `display.dw/dh`，1 CSS px = 1 canvas px，不需要缩放。

**画笔配置改为面板内 slider**：
- 从 toolbar mask group 移除 `image.maskConfig.size`（大小 dropdown）和 `image.maskConfig.color`（颜色 dropdown）
- `ImageMasker.vue` 底部新增画笔配置面板：size slider (2-120px)、opacity slider (5%-100%)、颜色色块按钮、橡皮擦切换
- 面板用本地 `reactive<MaskConfig>`（`brush`），`@input` 立即渲染反馈，`@change` 同步到父组件 node data
- 父组件通过 watcher 监听 props.maskConfig 将外部改动（toolbar 橡皮擦）同步回本地 brush
- 新增 `@update:mask-config` emit，ImageNode.vue 通过 `onMaskConfigUpdate` 写回 node data
- 移除死代码 `handleImageMaskConfig` 函数和 `image.maskConfig` 命令注册
- 移除未使用图标 `penSvg`、`colorSvg`
