## UiTestPage.vue TS2307 模块解析修复

**问题**: `vue-tsc --project tsconfig.app.json` 报:
- `TS2307: Cannot find module '@mini-canvas/canvas-core/components/Ui'`
- `TS2307: Cannot find module '@mini-canvas/canvas-core/components/Ui/types'`

**根因**: `canvas-core/package.json` exports 野卡 `"./*": "./src/*"` 在 TS strict project mode 无法解析目录型子路径。

**修复**: 添加显式 subpath exports + `types` 字段:
```json
"types": "./src/index.ts",
"exports": {
  ".": { "types": "./src/index.ts", "default": "./src/index.ts" },
  "./components/Ui": { "types": "./src/components/Ui/index.ts", "default": "./src/components/Ui/index.ts" },
  "./components/Ui/types": { "types": "./src/components/Ui/types.ts", "default": "./src/components/Ui/types.ts" },
  "./*": "./src/*"
}
```
验证: `vite build` 通过，`vue-tsc --project tsconfig.app.json` 中 UiTestPage 错误已消除。

## MentionMenu "无匹配结果" 修复 & showMenu 过滤覆盖 bug

**用户反馈**: 输入框中 @ 提及菜单总显示"无匹配结果"，无数据时不应显示此提示。

**根因**:
1. `showMenu()` 无条件用 `unref(props.variables/resources) || []` 覆盖 `filteredVariables/filteredResources`，导致 dispatchTransaction 中的过滤结果被吞掉
2. `textBefore.endsWith("@")` 分支调用 showMenu 前未设置 filtered 列表

**修复**:
- `MentionMenu.vue`: 删除"无匹配结果"空状态模板和 `.empty-message` CSS
- `useEditor.ts`:
  - `showMenu` 不再覆盖 filtered 列表（调用方负责在调用前设置）
  - `textBefore.endsWith("@")` 分支增加 `filteredVariables/filteredResources` 赋值

**验证**: prosemirror-editor-bundle 和主项目 `vite build` 均通过。

## ProseMirrorEditor 无法输入 — Vue 双实例问题

**现象**: ProseMirrorEditor 在 ImageBottomToolbar 中无法输入，控制台报:
- `Missing ref owner context. ref cannot be used on hoisted vnodes` (MentionMenu)
- `withDirectives can only be used inside render functions`

**根因**: `prosemirror-editor-bundle` 的 `vite.config.ts` 中 `rollupOptions.external` 为空数组，导致 library build 将 Vue 运行时打入 bundle。消费者 (mini-canvas) 也有自己的 Vue 实例，运行时存在两个 Vue 实例，ref/directive 上下文不一致。

**修复**:
1. `vite.config.ts`: non-webcomponent 模式下 externalize `vue` 及所有 prosemirror-* 包
2. `vite.config.ts`: webcomponent 模式下 `emptyOutDir: false`，避免覆盖主构建产物
3. 验证: dist 同时包含 es.js (34KB) + umd.js (24KB) + standalone.js (216KB)
