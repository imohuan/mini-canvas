## 修复：nodesById 异步延迟导致 isValidConnection 误判

**错误**：`getInvalidConnectionReason` 报告 `node not found in nodesById`，但 `getNodes` 中确实存在该节点。导致连接校验错误返回 `invalid`，拖线时出现"无法连接"误提示。

**根因**：`useCanvasConnection.ts:268` `nodesById` 用 `shallowRef + watch` 维护，watch 回调异步执行。当 `getNodes` 新增节点后，watch 尚未触发，`nodesById` 仍然持有旧 Map，导致 O(1) 查找 miss。

**修复**：将 `nodesById` 改为 `computed`，保证与 `getNodes` 同步：
```ts
// Before: shallowRef + watch（异步延迟）
const nodesById = shallowRef(new Map<string, Node>())
watch(() => getNodes.value, ...)

// After: computed（同步）
const nodesById = computed(() => {
  const map = new Map<string, Node>()
  for (const n of getNodes.value as Node[]) map.set(n.id, n)
  return map
})
```
`computed` 在依赖变化时立即重新计算，`.value` 访问始终拿到最新数据。
