# 2026-06-27 工作日志

## 修复：拖拽连接线时 BaseNode 递归更新错误

**错误**：`Maximum recursive updates exceeded in component <BaseNode>` — 拖拽连接线时 Vue 检测到递归更新超过上限。

**根因**：`useCanvasConnection.ts` 的 `buildConnectionEdgeProps` 函数在 Vue render 阶段写入 `canvas.connectionState.hoverNode`（Pinia reactive state）。该函数在 `<template #connection-line>` 插槽中调用，属于 render 阶段。写入触发的 reactive 依赖传播回自身，形成 `render → write → re-render → ...` 递归链。

**修复（两处）**：

1. **`useCanvasConnection.ts:780-794`** — 将 `hoverNode` 写入包装在 `nextTick` 中，迁出 render 阶段：
   ```ts
   if (hoverChanged) {
     const nextHover = ... // 捕获快照值
     nextTick(() => { canvas.connectionState.hoverNode = nextHover })
   }
   ```

2. **`BaseNode.vue:273-306`** — `cardInlineStyle` 从 `computed`（每次返回新对象）改为 `shallowRef` + `watch`（值变化时才更新引用），减少不必要的样式重绘。放置位置在 `cardTransform` 定义之后，确保依赖的 `cardWidth`、`cardHeight`、`zoom`、`cardTransform` 已声明。
