### Bug 调查：text→image 连接 (commit 546f2a8)
- 用户报告"文本节点无法连接图片节点"

- text 节点有 source handle（`hasSource: true`），image 节点有 target handle（`hasTarget: true`）
- 可能原因：用户尝试的是 image→text（text 无 target handle，canReceiveInput:false），或浏览器缓存旧代码
- `acceptsInputs?: string[]` 在 CanvasNodeDefinition 接口，undefined=接受全部（向后兼容）
- `getInvalidConnectionReason` 返回中文提示如"360全景不接受视频输入"，复用 BaseNode.vue 的 invalid-connection-tooltip 显示
- `isValidConnection` 和 `getInvalidConnectionReason` 各自独立加校验（前者给 VueFlow 用，后者给 hover 反馈用）
- Canvas.vue 传 `getNodeDefinition: (type) => nodeRegistry.get(type)` 闭包

### Bug 修复：ConnDebug 日志刷屏卡死 (commit f92a5db)
- **根因**：`getInvalidConnectionReason` 在 missing port / canonical / cycle / node not found 等分支都有 `console.log`，拖线时每帧调用导致刷屏卡死
- **次要问题**：text→text 连接本应被类型校验拦截，但 port 检查在类型校验之前，text 无 targetPosition 永远先走 missing port 分支打日志
- **修复**：删除全部 ConnDebug console.log；类型校验提前到 port 检查之前
- **教训**：高频调用路径（拖线 onMouseMove）禁止 console.log，用 debugLog 替代（受 DEBUG_CONNECT 开关控制）

### Bug 修复：拖线卡死（render 阶段 reactive write 循环）(commit 16bc835)
- **根因**：`buildConnectionEdgeProps` 在 connection-line slot **render 阶段**执行，内部用 `nextTick` 写入 `canvas.connectionState.hoverNode` → hoverNode 变化 → BaseNode computed 重算 → BaseNode 重渲染 → connection-line slot 重渲染 → `buildConnectionEdgeProps` 再次执行 → 再注册 nextTick → **Maximum recursive updates exceeded 死循环**
- **次要**：`getInvalidConnectionReason` 对同一 candidateConnection 调用两次（检查 + 获取消息），浪费 wouldCreateCycle 计算
- **修复**：`nextTick` → `requestAnimationFrame` + `hoverRafId` 标志位（rAF 在下一帧执行，不形成 microtask 循环；标志位确保每帧最多一个 rAF）；缓存 `getInvalidConnectionReason` 结果
- **教训**：render 阶段（slot 渲染函数、computed、template 表达式）禁止写入 reactive state，即使用 nextTick 也不行 —— nextTick 在 microtask 阶段执行，仍可能触发同帧重渲染循环。用 `requestAnimationFrame` 确保写入发生在下一帧
