## AlignGuidePlugin 添加 Panel 开关

- **文件：** `src/canvas/core/plugins/align-guide/AlignGuidePlugin.ts`
- **变更：** 通过 `context.panels.registerSetting` 注册 `type: 'boolean'` 的 `align-guide.enabled` 设置项，用 `context.store.toRef('enabled', true)` 绑定为响应式 `enabledRef`
- **开关逻辑：** `nodeDragStart` 和 `nodeDrag` 入口处检查 `enabledRef.value`，为 false 时跳过全部对齐计算和吸附
- **模式：** 与 AlignArrangePlugin（gap toRef）和 MiniMapPlugin（visible toRef）一致

## 修复 ImageCropper 裁剪框 resize 弹回 + 不显示问题

**根因（systematic-debugging 定位）：**
1. ImageCropper.vue 从未被渲染（ImageNode.vue 没 import 它）
2. resize 公式完全错误：`nw = Math.abs((scX + dx) - ax)` 漏了 `scW`，导致拖拽时宽度变成 `|dx|`（几像素），被 clamp 强制拉回 MIN_CROP=20 → "弹回"
3. `containerW/containerH` 用 side-effect ref 写入（在 wrapperStyle computed 内赋值），zoom 变化时 display.scale 过期

**修复：**
- `ImageNode.vue`：`import ImageCropper`，`v-if="isCropping"` 渲染，传入 nodeId/imageUrl/imageWidth/imageHeight
- `ImageCropper.vue` `onPointerMove` resize 逻辑重写：固定对边、移动当前边，`nw = scW + dx`（e/s）或 `nw = right - nx`（w/n）
- `containerW/containerH` 改为独立 computed
- `z-index` 从 +10 改为 +100
- 去掉 `overflow: hidden`（裁切手柄不被裁切）
- 测试文件：`crop-test.html`（含自动化断言）
