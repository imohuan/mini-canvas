## CustomNode.vue v-bind="$props" 导致 position prop 类型冲突
- 报错：`Invalid prop: type check failed for prop "position". Expected String, got Object`
- 根因：`CustomNode.vue` 用 `v-bind="$props"` 把 VueFlow `NodeProps`（含 `position: {x,y}`）全量传给子组件。`StageBottomToolbar` 没声明 prop，`position` 对象变成 fallthrough attr 穿透到 `NodeToolbar`，覆盖了显式的 `:position="Position.Bottom"`（字符串）
- 第一次修复：只传 `:id` 和 `:data` → 导致新报错 `Missing required prop: "events"`（ImageNode 需要完整 NodeProps）
- 最终修复：`src/canvas/core/components/CustomNode.vue` 新增 `childProps` computed，排除 `position` 但保留其余所有 NodeProps 字段
- 验证：`vue-tsc -b --noEmit` 零错误通过
