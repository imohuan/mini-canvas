## 拖拽端口连接线失败 Bug 修复

- **Bug 1 (核心)**: `onConnectEnd` finally 块中 `suppressHandles = true` 应为 `false`。导致所有走 onConnectEnd 路径的连接（snap 吸附、body hit、拖到空白）结束后端口永久隐藏，后续无法再拖线。
- **Bug 2**: `getInvalidConnectionReason` 缺少 `sourcePosition`/`targetPosition` 检查，与 `isValidConnection` 不一致。拖拽时可能显示"有效"反馈，释放后因 position 缺失静默失败。
- **FileSystemAdapter InvalidStateError**: handle 缓存失效时项目目录层 handle 每次写都重新获取 root handle，容易触发。改为缓存项目目录 handle，减少 root handle 调用。InvalidStateError 时清除缓存并在 StoragePlugin 中断开 fsAdapter，后续只走 localStorage。
- 添加 [ConnectDebug] 系列诊断日志到 onConnectStart/onConnect/onConnectEnd/createConnection 关键路径，方便复现验证。
- 文件变更: useCanvasConnection.ts, FileSystemAdapter.ts, StoragePlugin.ts
