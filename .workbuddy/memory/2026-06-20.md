## Bug修复：右键菜单创建节点 type 带 create: 前缀
- **文件**: `src/canvas/core/Canvas.vue` - `createNodeFromMenuItem` 函数
- **问题**: `MenuRegistry.createNodeItems()` 给 nodeType 加 `create:` 前缀作为菜单项 id（如 `create:text`），但 `createNodeFromMenuItem` 直接用 `item.id` 设置 `node.data.nodeType`，导致节点 type 变成 `create:text`
- **影响**: 除了 nodeType 错误外，`nodeRegistry.getDefaultSize()` 和 `canReceiveInput()` 也查不到对应定义，`resizable` 判断也失败
- **修复**: 在函数开头从 `item.id` 去掉 `create:`/`connect:` 前缀得到真实 `nodeType`，所有后续引用改用 `nodeType`；同时 `resizable` 改用 `nodeRegistry.isResizable(nodeType)`
- **修复策略**: 从 git HEAD 恢复具有正确中文内容的版本，然后应用 `state` 重构的代码变更（`canvas.state.XXX` → `canvas.state.core.XXX`）
- **关键变更**:
  - 5 个简单文件：git HEAD 恢复 + 全局 `canvas.state.` → `canvas.state.core.` 替换
  - useCanvasStore.ts：手动重构 `state` 结构（扁平 → `core:` 嵌套），同步更新 serializer 支持新旧两种持久化格式的兼容
