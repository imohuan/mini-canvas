## Fix: AlignGuidePlugin 组拖拽时忽略组内节点

**问题：** 拖拽组（Group）时，对齐辅助线计算未排除组内部的子节点。子节点 `computedPosition` 会被错误用作对齐参考，产生无意义的辅助线。

**修复：** `AlignGuidePlugin.ts` 第 160 行，`others` 过滤条件增加 `n.parentNode !== draggedNode.id`，排除被拖拽组的直接子节点。

**影响文件：** `src/canvas/core/plugins/align-guide/AlignGuidePlugin.ts`

## 修正 content-clip counter-scale 影响所有节点内容的问题

- **根因**: `BaseNode.vue` 的 `cardFrameStyle` 对 `custom-node-content-clip` 做了 `transform: scale(1/zoom)` counter-scale，所有节点（Text/Image/Video）内容都被自动抵消缩放
- **最终方案**: 将 `<slot name="content">` 移出 `custom-node-content-clip`，和 clip 平级。clip 仅承载背景+边框（通过内部子 div 应用 counter-scale），content 不受影响，自然跟随画布缩放
- **结构**:
  ```
  custom-node-card
    ├── custom-node-content-clip (overflow:hidden 裁剪)
    │     └── div:cardFrameStyle (scale(1/zoom) 仅作用于背景/边框)
    └── slot:content (不受 counter-scale，自然跟随 VueFlow zoom)
  ```
- **改动**:
  1. `BaseNode.vue`: content slot 移出 clip，cardFrameStyle 移到 clip 内部子 div；移除 `text-content` slot
  2. `CustomNode.vue`: 恢复统一 `node` 字段路由，移除 `content`/`textContent` 拆分
  3. `TextNode.vue`: 固定 `absolute inset-0 p-4` + `text-sm`，无任何 zoom 逻辑

## AlignGuidePlugin 闪烁修复 + 日志前缀统一

- **根因**: `renderLines()` 每次用 `el.innerHTML = html` 全量重建 DOM，拖拽时 30-60fps 调用导致严重布局抖动/闪烁
- **修复**:
  1. 预创建 2 个固定 DOM 元素（vLine/hLine），首次 `ensureLineElements()` 创建后只通过 CSS left/top/display 更新位置和显隐，零 DOM 重建
  2. `lastGuideSignature` 签名比较跳过与上一帧相同的渲染，减少无效回流
  3. `clearLines()` 改为 `display:none` 隐藏而非 `innerHTML=''` 销毁
- **日志**: 所有 logger 调用统一加 `[AlignGuide]` 前缀；renderLines 增加带序号和详细坐标的渲染日志
- **影响文件**: `src/canvas/core/plugins/align-guide/AlignGuidePlugin.ts`

## AlignGuidePlugin 吸附振荡根因修复（锁 dimensions — 替代粘性方案）

- **上一轮 Hysteresis 方案失败原因**: 粘性只锁定了吸附目标，但 `node.dimensions` 每帧仍在波动，导致同一目标下每帧算出的 snap delta 也不同（2.96 vs 7.41），反而加剧抖动
- **业界调研结论**: tldraw 的做法是每帧从原始鼠标位置起算 nudge，不累积。但我们的问题是拖拽节点自身 `node.dimensions` 在事件间波动（counter-scale / content reflow）
- **最终方案**: `nodeDragStart` 时锁定 `dragDimensions = { width, height }`，`calculateGuides` 使用锁定值构造 `db` 边界框，不再读 `draggedNode.dimensions`。`nodeDragStop` 释放
- 这确保整个拖拽过程中 `db` 的 9 个 edge pair 与各目标节点的距离关系恒定，只随鼠标位置变化而移动，不会因为尺寸波动跳到另一个 edge pair
- 回滚了上一轮的 Hysteresis 粘性代码
- **影响文件**: `src/canvas/core/plugins/align-guide/AlignGuidePlugin.ts`

## 22:51 — MultiSelectPlugin: 修复 group 子节点框选坐标不匹配

**Bug**: `isNodeInRect()` 直接用 `node.position`（group 子节点为相对父 group 坐标）与 flow 绝对坐标 `rect` 做碰撞检测，坐标系不匹配 → 选中节点错误 → 选框/移动行为错误。

**Fix**: `MultiSelectPlugin.ts:124-125` → `node.position.x/y` 替换为 `node.computedPosition?.x ?? node.position.x`（VueFlow 内置绝对坐标属性，无 parentNode 时回退）

**Verification**: vue-tsc type check passed, vite build passed. 1 file, 2 lines changed.
