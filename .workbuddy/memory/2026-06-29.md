## SelectionFrame Toolbar 偏移随 zoom 变化 Bug 修复

**问题**: 多选时 BaseToolbar → NodeToolbar 的 toolbar 偏移随画布缩放变化，与单节点行为不一致。

**根因**: `selectionFramePaddingTop` 是 canvas 空间像素值，但被直接作为 `extraOffset` 传给 NodeToolbar 的 `offset`（screen 空间）。`getTransform` 中 offset 不乘 zoom，导致 gap = `topToolbarOffset + paddingTop * (1 - zoom)`，随 zoom 变化。

**修复**: `SelectionFrame.vue:276`
```diff
- :extra-offset="canvas.state.core.selectionFramePaddingTop"
+ :extra-offset="canvas.state.core.selectionFramePaddingTop * viewport.zoom"
```

**验证**: 浏览器实测 3 个 zoom 级别，修复后 gap 恒定 12px（= topToolbarOffset），与单节点行为一致。

**关键教训**: 之前 3 轮分析只停留在代码阅读层面，从未实际在浏览器中验证。systematic-debugging 要求"收集证据"——用 playwright 在运行时测量 DOM 位置，一次就确认了根因。

## Ctrl+D 快捷键被浏览器拦截修复

**问题**: 画布 Ctrl+D 复制一份节点，实际触发了浏览器收藏书签。

**根因**: `ClipboardPlugin.ts:306` 的 handler `() => { performCopy() && performPaste() }` 用了 block body，永远返回 `undefined`。`ShortcutManager` 用 `=== true` 严格判断 handler 返回值才调用 `preventDefault()`，因此浏览器默认行为未被阻止。

**修复**: 去掉 `{}` → `() => performCopy() && performPaste()`，expression body 直接返回 boolean 结果。

## 右键菜单功能不生效修复

**问题**: 右键菜单中的"复制"/"复制一份"/"删除节点"/"删除连线"点击无效。

**根因**: `ContextMenuPlugin.onMenuSelect` 只处理"创建节点"和"连线+创建节点"两种场景，完全忽略了 `MenuItemDefinition.commandId`，从未调用 `context.commands.execute()`。`resolveItems` 也没有传递 `commandId` 到 resolved item 中。

**修复** (3 个文件):
1. `types.ts` — `CanvasMenuItem` 增加 `commandId?: string`
2. `ContextMenuPlugin.ts` — `ResolvedMenuItem` 增加 `commandId`，`resolveItems` 传递 `commandId`
3. `ContextMenuPlugin.ts` — `onMenuSelect` 优先检查 `item.commandId`，有则调用 `context.commands.execute()` 并 close

**注意**: builtinMenuItems 中注册的命令使用闭包捕获的 `ctx`（PluginContext）而非 `CommandContext` 参数，因此 `context.commands.execute(id)` 不传 context 也能正常工作。

## 蒙版入口点击后 toolbar 不显示修复

**问题**: 单击"蒙版"按钮后 mask 组按钮不出现。

**根因**: `ToolbarButton.vue:87` — 按钮 `@click` 无 `.stop` 修饰符。点击事件冒泡到 Vue Flow viewport，触发 pane click → 取消选中节点。`NodeToolbar.isActive` 检查 `nodes[0].selected && getSelectedNodes.length === 1`，节点取消选中后 `isActive = false`，整条 toolbar 消失。此时 `_toolbarGroup` 虽已设为 `'mask'`，但工具栏已不可见。

**修复**: `@click` → `@click.stop`，阻止冒泡。
